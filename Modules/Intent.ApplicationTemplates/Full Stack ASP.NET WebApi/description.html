A full technology stack for building a web server using ASP.NET WebApi 2.2 in Visual Studio.

This template configures an N-Layered architecture with a rich domain based on Domain Driven Development (DDD) principles.

The following technologies are used:

  • ASP.NET WebApi 2.2 - framework for building HTTP services
  • OWIN - middleware framework between the web server and web application
  • Entity Framework - Microsoft's recommended ORM (Object Relational Mapper), providing an abstraction on the database.
  • Unity - dependency injection framework.

Other notes:

  • Service calls are implemented using a lazy dispatch pattern which is abstracted away from the WebApi technology and technical concernts. Your service call business code should be implemented in the 'SCH' classes.
  • A collection oriented repository pattern provides access to the udnerlying database via Entity Framework. The DbContext backing each repository is isolated to each service call. The same instance is returned for any instantiation of a repository during the service call.
  • Dependency injection is implemented throughout each pattern in the application. Add additional registrations in thte UnityConfig.cs file.
